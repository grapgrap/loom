# user scope의 자동 참조를 제거하고 보관소로 전환한다

## 맥락

기존에는 모든 스킬의 Step 0에서 `~/.claude/.loom/principles/index.md`(user scope)를 자동 로드하여 project scope와 병합했다. 이는 사용자의 사고 양식이 모든 프로젝트에 자동 적용되어야 한다는 가정에서 출발했다.

원칙·룰·컨벤션 분화가 진행되면서 정체성 충돌이 드러났다. 원칙(1층)은 도메인 무관하므로 user scope에 자연스럽지만, 컨벤션(3층)은 도구·언어 종속적이라 프로젝트마다 다른 형식이어야 한다. user scope에 셋이 모두 들어가면 컨벤션 같은 환경 종속 항목이 모든 프로젝트에 강제된다.

## 결정

user scope의 자동 참조를 제거한다. user scope (`~/.claude/.loom/`)는 보관소로 남기되, 스킬은 더 이상 자동 로드하지 않는다. 사용자가 명시적으로 호출할 때만 사용한다.

미래에 명시적 extends 메커니즘(eslint shared config 같은 형태)을 도입할 가능성을 열어둔다. user scope에 보관된 데이터는 그 시점에 활용 가능하다.

### 전제 조건

- 사용자가 자기 사고 양식을 명시적으로 가져올 의지가 있다: 자동 적용이 없어도 필요할 때 직접 가져올 수 있다
- user scope의 데이터는 보존된다: 폐기가 아니라 자동 참조만 끊는다

### 제약 조건

- 기존 사용자의 user scope 데이터를 손실 없이 보존한다: 13개 항목 그대로 유지
  - 제약이 없었다면: user scope 자체를 폐기하여 단순화했을 것

## 대안

### 자동 참조 유지

- 설명: 기존 scope 해소 메커니즘 유지. user → project 자동 병합.
- 트레이드오프: 컨벤션(3층) 같은 환경 종속 항목이 user scope에 들어갈 때 모든 프로젝트에 강제되는 정체성 충돌 발생.

### user scope 완전 폐기

- 설명: user scope 데이터까지 모두 제거하고 project scope만 사용
- 트레이드오프: 기존 13개 사용자 데이터 손실. 명시적 extends를 도입할 미래에 활용할 자료도 사라짐.
