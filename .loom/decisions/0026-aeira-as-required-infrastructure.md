# aeira를 loom의 필수 탐색 인프라로 도입한다

## 맥락

loom의 `.loom/` 문서들은 위키링크로 관계를 표현하며, 스킬들이 맥락을 로드할 때 관련 문서를 탐색해야 한다. 기존에는 grep으로 수행했으나, 문서가 비대해지면서 토큰 경제성과 탐색 정확도가 떨어졌다.

decision-0014에서 문서 탐색 도구를 선택적으로 제공하려 했으나 폐기되었다. 당시 전제는 "사용자가 자신의 도구를 제공할 것이고, loom은 그것을 선택적으로 사용한다"였다. 현재는 loom 문서의 비대화로 최적화된 탐색이 선택이 아닌 시스템 요구사항이 되었다.

aeira(grapgrap/aeira)는 마크다운 위키링크 관계를 방향 그래프로 구축하여 CLI에서 탐색하는 도구이다.

## 결정

aeira를 loom의 필수 인프라로 도입하여, 스킬들이 맥락 로드 시 직접 활용한다.

- 스킬의 Step 0에서 `aeira search`로 진입점을 확보하고, `aeira graph neighbors`로 관계를 탐색한다
- 문서를 변경하는 스킬은 변경 후 `aeira sync`를 강제 호출한다
- 그래프 대상은 `.loom/` 내 문서로 한정한다
- aeira 설치는 사전 요구 조건이다

### 제약 조건

- loom 문서의 비대화가 grep 기반 탐색의 한계를 넘었다: 구조적 탐색 도구가 필수

### 전제 조건

- aeira가 설치되어 있다: 별도 번들링 없이 사전 설치를 전제한다
- loom 문서가 위키링크로 관계를 표현한다: aeira의 그래프 구축이 유효하다

## 대안

### 선택적 제공 유지 (decision-0014 방식)

- 설명: 도구를 선택적으로 제공하고 없으면 grep 폴백
- 트레이드오프: 문서 비대화 문제를 해결하지 못한다. 폴백 경로가 존재하면 최적화 도입이 계속 미뤄진다

### MCP 서버로 분리

- 설명: aeira를 MCP 서버로 감싸 스킬과 간접 연결
- 트레이드오프: 간접 레이어가 추가되지만, CLI 직접 호출로 충분한 상황에서 불필요한 복잡도
