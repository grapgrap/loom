# Index 갱신은 자동이 아닌 판단 기반으로 한다

## 맥락
concept 문서가 변경될 때 index를 어떻게 갱신할지 논의했다. 자동 갱신과 판단 기반 갱신 중 선택이 필요했다.

## 결정
concept 문서가 변경될 때마다 index 갱신이 필요한지 판단한다. 무조건 갱신하지 않는다.

## 대안
- 자동 갱신 — concept 변경 시 항상 index를 업데이트. 단순하지만, 사소한 수정(오타, 문구 다듬기)에도 index를 건드리게 되어 불필요한 변경이 발생한다.

- shape, review 스킬에 판단 로직 추가
