# Index 갱신은 자동이 아닌 판단 기반으로 한다

## 맥락

concept 문서가 변경될 때 index를 어떻게 갱신할지 논의했다.
자동 갱신과 판단 기반 갱신 중 선택이 필요했다.

## 결정

concept 문서가 변경될 때마다 index 갱신이 필요한지 판단한다. 무조건 갱신하지 않는다.

### 제약 조건

- index는 탐색 지도 역할을 한다: 사소한 변경에도 지도를 다시 그리는 것은 비효율적이다

### 전제 조건

- 갱신이 필요한 변경과 불필요한 변경이 구분 가능하다: 새 엔티티 추가, 관계 변경, 책임 변경은 갱신이 필요하고, 오타나 문구 다듬기는 불필요하다

## 대안

### 자동 갱신

- 설명: concept 변경 시 항상 index를 업데이트한다
- 트레이드오프: 단순하지만, 사소한 수정(오타, 문구 다듬기)에도 index를 건드리게 되어 불필요한 변경이 발생한다
