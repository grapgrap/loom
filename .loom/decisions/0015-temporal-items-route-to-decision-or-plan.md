# 시점 종속 항목을 decision 또는 plan으로 라우팅한다

## 맥락

shape이 concept에 시점 종속 항목("아직 결정되지 않았다", "현재 지원하지 않는다" 등)을 기록하는 문제가 반복적으로 발생했다 (GitHub 이슈 #1, 위반 5건).
concept은 영속 문서이므로 시점 종속 항목이 들어가면 안 되지만, shape 시점에 미결정 사항을 수용할 구조가 없어 concept의 트레이드오프 섹션에 흡수되었다.

## 결정

시점 종속 항목을 식별하면 concept이 아닌 시점 종속 문서로 라우팅한다:
- concept의 형태에 영향을 준 보류 → decision에 기록 (보류 결정)
- 나중에 해야 할 작업 → plan의 태스크로 이동

기존의 금지 규칙("concept에 넣지 않는다")을 양의 행동("decision 또는 plan으로 보낸다")으로 전환한다.

### 제약 조건

- concept은 영속 문서다: 세션이 끝난 후에도 유효해야 한다
- decision과 plan은 시점 종속 문서다: 특정 시점의 맥락과 범위를 기록한다

### 전제 조건

- 금지만으로는 에이전트의 기록 압력을 해소하지 못한다: 미결정 사항을 식별했을 때 어디로 보내라는 양의 방향이 필요하다
- "보류 결정"은 concept의 형태에 영향을 주는 구조적 선택이다: decision의 기록 범위에 포함된다

## 대안

### 미결정 사항을 기록하지 않는다

- 설명: concept의 여백으로 존재하게 두고, 필요할 때 plan에서 자연스럽게 떠오르게 한다
- 트레이드오프: 보류의 맥락(왜 보류했는지, 어떤 대안이 있었는지)이 유실된다

### 새로운 문서 유형을 만든다 (e.g., deferred-topics)

- 설명: 미결정 사항 전용 문서 유형을 도입한다
- 트레이드오프: 기존 엔티티(decision, plan)가 이미 시점 종속 문서로서 이 역할을 수행할 수 있으므로 불필요한 복잡성이다
