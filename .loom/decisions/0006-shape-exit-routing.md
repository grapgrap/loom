# shape 완료 후 라우팅 판단을 추가한다

## 맥락

shape → plan → 구현 → review 흐름에서, shape 완료 후 plan이 유일한 출구였다.
엔터프라이즈급 프로젝트 마이그레이션에서 단일 작업이나 기계적 변환만 남는 경우가 발생했고, 이때 plan 작성은 오버헤드였다.

## 결정

shape의 Summary 뒤에 "완료 후 라우팅" 섹션을 추가하여, plan 진행 / 직접 실행 / shape 재진입의 세 경로를 공식화한다.

### 제약 조건

- shape의 triage에 이미 라우팅 패턴이 존재한다: 완료 후 라우팅도 동일한 패턴(표 + 사용자 동의)을 따라야 일관성이 유지된다

### 전제 조건

- shape의 산출물이 plan 없이도 실행 가능한 경우가 존재한다: 단일 작업이나 기계적 변환이 그 예이다

## 대안

### 기존 구조 유지, 비공식적으로 plan 건너뛰기

- 설명: 필요에 따라 암묵적으로 plan을 생략한다
- 트레이드오프: 암묵적 규칙은 일관성을 해치고, 판단 기준이 없어 매번 재논의가 필요하다
