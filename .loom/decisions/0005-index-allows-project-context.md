# index는 프로젝트 맥락(범례)을 직접 포함한다

## 맥락

index concept의 설계 방향이 "내용을 인라인하지 않고 링크로 참조한다"로 되어 있었다.
엔터프라이즈급 프로젝트(POP)에 최초 적용하면서, 프로젝트의 서비스 개요를 index에 포함해야 하는 상황이 발생했다. 기존 규칙은 이를 금지하는 것으로 읽혀, 원칙 위반으로 오판하는 일이 발생했다.

## 결정

"인라인하지 않는다"의 범위를 "개별 concept의 내용을 중복하지 않는다"로 한정한다. 프로젝트 맥락(정의, 범위, 핵심 기능)은 지도의 범례로서 index 자체의 역할에 포함한다.

### 제약 조건

- index의 책임에 "시작점 역할"이 포함되어 있다: 프로젝트 맥락 없이는 이 책임을 수행할 수 없다

### 전제 조건

- 프로젝트 맥락과 concept 내용의 구분이 가능하다: 프로젝트의 정의/범위/핵심 기능은 어떤 개별 concept에도 속하지 않는 메타 정보이다

## 대안

### 기존 규칙 유지, 프로젝트 맥락을 별도 concept으로 분리

- 설명: "프로젝트 개요" 같은 별도 concept을 만들어 맥락을 거기에 기록한다
- 트레이드오프: 프로젝트 맥락은 특정 concept에 속하지 않는 메타 정보이므로 위치가 부자연스럽다. index의 시작점 역할 책임과도 충돌한다
