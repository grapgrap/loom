# Auto memory 정제를 증류 이후에 배치한다

## 맥락

auto memory의 생명주기에 정제 경로가 없다.
생성(review의 reflect)과 소비(calibrate의 원칙 증류)는 존재하지만,
유효하지 않거나 과도하게 특수한 메모리를 정리하는 경로가 없어 제약이 단조증가한다.

calibrate Step 2에 auto memory 정제를 추가하면서,
정제와 증류의 순서를 결정해야 했다.

## 결정

증류를 먼저 수행하고, 정제를 나중에 수행한다.

순서: 기존 원칙 검증 → 원칙 증류 → auto memory 정제

### 제약 조건

- 증류는 반복 패턴 발견에 의존한다: 개별로는 무효해 보이는 메모리도 여러 개가 모이면 반복 패턴의 증거가 된다. 정제가 먼저 돌면 이 증거가 소실될 수 있다.

### 전제 조건

- 정제는 증류의 후처리다: 증류에서 승격된 메모리를 포함하여 중복/무효를 한 번에 정리할 수 있다.
- auto memory는 대부분 교정(부정) 피드백이다: 개별 메모리의 유효성 판단과 패턴 발견은 별개의 관점이며, 패턴 발견이 전체 증거를 필요로 한다.

## 대안

### 정제 먼저, 증류 나중 (A)

- 설명: 유효하지 않은 메모리를 걸러낸 후 깨끗한 입력에서 원칙 후보를 찾는다.
- 트레이드오프: 증류의 입력이 깨끗해지는 이점이 있으나, "현재 무관"으로 제거된 메모리가 반복 패턴의 일부였을 때 증거가 소실된다. 증류의 핵심 기능(반복 패턴 발견)을 훼손하는 위험이 이점보다 크다.
