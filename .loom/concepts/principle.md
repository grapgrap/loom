# Principle

경험에서 축적되는 범용적 교훈. 프로젝트를 넘어서 유효한 판단 기준이다.

## 목적

반복되는 실수를 방지하고 설계 방향의 일관성을 유지하는 가드레일을 제공한다.

### 핵심 시나리오

- shape에서 설계 방향을 수립할 때 principle을 참조하면 과거의 교훈이 반영된 판단을 할 수 있다
- review에서 구현을 검증할 때 principle에 어긋나는 부분을 식별할 수 있다

## 동작 원리

### 역할

- 판단 근거: shape, plan, review 전 단계에서 판단의 방향을 형성한다
- 가드레일: 반복되는 실수를 방지하는 설계 제약으로 작동한다

### 핵심 동작

- 사용자의 것이다 — 시스템이 임의로 삭제하지 않는다
- 두 가지 조건을 모두 충족해야 한다:
  1. **범위**: 다른 프로젝트, 다른 맥락에서도 유효한가
  2. **성격**: 설계(shape) 또는 검증(review)에서 판단의 근거로 작동하는가
- 성격의 리트머스 테스트: "몰랐다면 다르게 **결정**했을 것" → principle, "몰랐다면 **빼먹었을** 것" → 체크리스트

## 트레이드 오프

- 프로젝트 특수한 노하우는 principle이 아니라 해당 concept이나 decision에 기록한다
- 구현 과정의 절차적 주의사항, 도구 사용법, 작업 체크리스트는 principle이 아니다
