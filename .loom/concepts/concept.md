# Concept

프로젝트의 도메인 엔티티를 정의하는 영속 문서.

## 목적

프로젝트가 다루는 세계의 구성 요소를 명시하여, 설계와 구현의 판단 기준이 되는 참조점을 제공한다.

### 핵심 시나리오

- 새로운 참여자가 concept을 읽으면 프로젝트의 공유 어휘(ubiquitous language)를 이해할 수 있다
- shape에서 concept을 참조하면 설계 방향의 정합성을 검증할 수 있다

## 동작 원리

### 역할

- 도메인 엔티티의 정의와 경계: 이 시스템이 무엇을 다루는가를 추적한다
- 공유 어휘: 프로젝트 전체에서 동일한 용어를 동일한 의미로 사용하게 한다

### 핵심 동작

- 구현이 아니라 도메인을 기술한다 — 특정 기술이나 라이브러리가 아닌 What에 집중한다
- 영속적으로 유지된다 — 구현이 끝나도 프로젝트의 지식 자산으로 남는다

## 트레이드 오프

- 구현 세부사항(기술 선택, 라이브러리, 파일 구조)은 concept에 기술하지 않는다
- skill(shape, plan, review, meditate)은 concept으로 정의하지 않는다 — skill은 loom의 "코드"이지 도메인 엔티티가 아니다

## 관련 컨셉

- [[decision]] - concept의 변경 맥락을 기록한다
- [[plan]] - concept을 참조하여 태스크를 구성한다
- [[principle]] - concept 설계의 판단 기준이 된다
- [[index]] - concept의 목록과 관계를 탐색 지도로 제공한다
