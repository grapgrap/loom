# Index

디렉토리의 진입점으로서, 해당 범위의 항목과 관계를 요약하는 탐색 지도.

## 목적

새로운 참여자가 프로젝트를 파악할 수 있는 시작점을 제공한다.

### 핵심 시나리오

- 새로운 참여자가 index를 읽으면 프로젝트의 전체 구조와 탐색 방향을 파악할 수 있다
- concept이 추가/변경되면 index의 갱신 여부를 판단하여 지도를 최신 상태로 유지한다

## 동작 원리

### 역할

- 프로젝트 맥락(범례): 프로젝트의 정의, 범위, 핵심 기능을 지도의 범례로 포함한다
- 항목 목록: 디렉토리 내 항목의 한 줄 설명을 직접 제공한다
- 관계도: 항목 간 관계를 시각적으로 표현한다 (해당되는 경우)

### 핵심 동작

- 프로젝트 탐색의 지도 역할을 한다 — 지도를 읽기 위한 맥락은 index 자체의 역할로 포함한다
- `.loom/index.md`가 루트 index로서 전체 구조의 진입점이 된다
- 각 `.loom/` 하위 디렉토리는 자체 index를 가질 수 있다

## 트레이드 오프

- 개별 concept의 내용을 중복하지 않는다 — concept의 상세는 링크로 위임한다
- 갱신은 자동이 아닌 판단 기반이다 — 사소한 수정에는 index를 건드리지 않는다
