# Index

## 정의
디렉토리의 진입점으로서, 해당 범위의 항목과 관계를 요약하는 탐색 지도.

## 책임
- 디렉토리 내 항목의 목록과 한 줄 설명 제공
- 항목 간 관계를 시각적으로 표현 (해당되는 경우)
- 새로운 참여자가 프로젝트를 파악할 수 있는 시작점 역할

## 관계
- 각 `.loom/` 하위 디렉토리는 자체 index를 가질 수 있다
- [[concept]], [[decision]], [[plan]], [[principle]] 문서가 변경될 때 관련 index의 갱신 여부를 판단한다
- `.loom/index.md`가 루트 index로서 전체 구조의 진입점이 된다

## 설계 방향
index는 내용을 인라인하지 않고 링크로 참조한다. 개별 문서의 내용을 중복하면 동기화 부담이 생기므로, 항목의 존재와 관계만 보여주고 상세는 개별 문서에 위임한다.
