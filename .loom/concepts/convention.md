# Convention

합의된 표현의 통일.

## 목적

여러 가능한 형식 중 하나를 합의로 선택하여 일관성을 만든다. 어느 형식이든 통일되면 작동한다는 본질이 있다.

### 핵심 시나리오

- 코드 표현의 일관성이 필요할 때 (이름 규칙, 들여쓰기 등)
- 도구·언어 차원에서 형식의 통일이 필요할 때
- 프로토콜이나 인터페이스 형태의 통일이 필요할 때

## 동작 원리

### 정체성

컨벤션은 "이 형식으로 통일한다"의 형태를 가진다.
원칙이 "왜"의 정당성, 룰이 "조건-행동"의 영역적 적용이라면, 컨벤션은 "합의된 한 형식"이다.

컨벤션은 본질적으로 임의 선택의 통일이다. 절대적 정당성보다 합의된 일관성이 핵심이다.

### 핵심 동작

- 두 가지 조건을 충족해야 한다:
  1. **반복성**: 해당 형식이 반복적으로 등장하는가
  2. **합의성**: 어느 형식이 통일되어야 한다는 합의가 형성되는가 (당위성보다 합의가 핵심)
- 도구·언어·맥락에 종속적이다 — 같은 형식이 다른 환경에서 다른 컨벤션이 될 수 있다
- 사용자의 것이다 — 시스템이 임의로 삭제하지 않는다
- user와 project의 `.loom/principles/`에 보관 가능하다

## 트레이드 오프

- 정당성이 명확한 형식은 컨벤션이 아니라 룰이나 원칙으로 표현한다
- 일회성 합의는 아직 컨벤션이 아니다 — 반복이 확인될 때 축적한다
- 컨벤션은 가장 가벼운 layer다 — 도구나 언어 변경 시 자연스럽게 갱신된다
