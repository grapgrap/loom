# Flow

여러 도메인을 가로지르며 시간 위에서 합성하는 경로를 기술하는 영속 문서.

## 목적

concept이 도메인을 정적으로 정의한다면, flow는 그 도메인들이 어떤 순서로 맞물려 하나의 완결된 일을 이루는지를 기술한다. 분해(concept)와 짝을 이루는 조합의 산물로서, 도메인 간 협력을 정렬의 공유 맥락으로 남긴다.

## 책임

하나의 트리거에서 시작해 결과에 이르는 해피 패스를, 도메인과 도메인 사이의 상호작용 순서(단계)로 기술한다. 각 단계가 어느 도메인의 관여인지를 드러냄으로써, flow 자체가 도메인들의 경계를 비추는 렌즈가 된다.

## 정책

- 단계는 도메인 경계를 넘는 마디 단위이며, 한 도메인 내부의 구현은 담지 않는다.
- 현재 상태의 스냅샷으로 동작한다 — 변경의 역사는 [[decision]]이 진다.
- 프랙탈로 합성된다 — 한 단계가 하위 흐름을 가리킬 수 있고, 한 자리에 여러 대안 흐름(변이)이 올 수 있다.

## 경계

- 도메인의 정적 정의는 다루지 않는다 — [[concept]]의 몫이다.
- 구현의 알고리즘·자료구조·라이브러리는 다루지 않는다 — 코드의 몫이며, 그 설계 판단은 [[decision]]에 남는다.
- 실행 작업의 분해와 순서는 다루지 않는다 — plan의 몫이다.

## 관계

- [[concept]] - flow가 가로지르는 도메인들. 분해(concept)와 조합(flow)으로 대등한 축을 이룬다.
- [[decision]] - flow의 변경 맥락이 기록된다.
- [[plan]] - 실행 시 flow가 태스크 구성의 참조가 된다.
