# Plan

## 정의
설계를 실행 가능한 태스크와 의존관계로 구조화한 문서. 무엇을 어떤 순서로 할 것인지를 명시한다.

## 책임
- 요구사항을 독립적으로 실행 가능한 태스크 단위로 분해
- 태스크 간 의존관계와 병렬 가능성을 명시
- 구현의 진행 상황을 추적하는 체크리스트 제공

## 관계
- [[concept]]을 참조하여 범위를 한정한다
- [[decision]]을 참조하여 설계 의도와 제약을 이해한다
- review가 plan의 충족 여부를 검증한다

## 설계 방향
plan은 산출물이지 실행이 아니다. plan 자체는 코드를 만들지 않는다. concept이나 decision과 연관되지만 독립적으로 존재한다 — 하나의 concept에서 여러 plan이 나올 수 있고, plan은 실행 단위로서 독자적 생명주기를 갖는다.
