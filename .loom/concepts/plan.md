# Plan

설계를 실행 가능한 태스크와 의존관계로 구조화한 문서.

## 목적

무엇을 어떤 순서로 할 것인지를 명시하여, 구현의 실행과 진행 추적을 가능하게 한다.

### 핵심 시나리오

- shape이 완료되면 plan으로 태스크를 분해하여 실행 순서와 의존관계가 명확해진다
- 구현 중 plan의 체크리스트로 진행 상황을 추적할 수 있다

## 동작 원리

### 역할

- 태스크 분해: 요구사항을 독립적으로 실행 가능한 단위로 나눈다
- 의존관계: 태스크 간 선후관계와 병렬 가능성을 명시한다
- 진행 추적: 구현의 진행 상황을 체크리스트로 제공한다

### 핵심 동작

- 산출물이지 실행이 아니다 — plan 자체는 코드를 만들지 않는다
- concept이나 decision과 연관되지만 독립적으로 존재한다
- 하나의 concept에서 여러 plan이 나올 수 있다

## 트레이드 오프

- 단일 작업이나 기계적 변환에는 plan을 작성하지 않는다 — shape의 완료 후 라우팅에서 판단한다
- plan은 독자적 생명주기를 갖는다 — concept의 수명과 결합되지 않는다
