# loom

요구사항을 설계로, 설계를 실행으로 구체화하는 Claude Code 플러그인.

[English](README.md) [한국어](README.ko.md)

## 왜 만들었는가

소프트웨어를 만들 때 우리는 이미 자연스러운 흐름을 따릅니다.
요구사항을 구체화하고, 계획을 세우고, 구현하고, 돌아보는 것. loom은 이 흐름을 구조화하여 재현 가능하게 만듭니다.

구조화된 흐름에서 나오는 산출물(설계 문서, 결정 기록, 원칙)은 시간이 지나도 유효해야 합니다.
loom은 피드백 루프를 통해 이 산출물들이 코드와 함께 살아있는 상태를 유지하도록 합니다.

## 워크플로우

loom은 다섯 가지 스킬로 동작합니다.

### 작업 흐름

**shape** -> **plan** -> **task** -> 구현

- **shape**: 모호한 요구사항을 설계 수준으로 구체화합니다. 접근 전략, 제약, 트레이드오프를 다루며, concept과 decision을 산출합니다. 전제·제약이 흔들렸으나 기존 설계를 유지한 경우 유지 decision을 생산합니다.
- **plan**: shape의 산출물을 실행 가능한 태스크와 의존관계로 분해합니다.
- **task**: plan의 태스크를 사용자와의 실행 합의를 거쳐 구현합니다. 방향성 초안을 제시하고 논의한 후 코드를 작성합니다.

### 피드백 루프

- **review**: 세션 단위의 검증입니다. 구현이 설계를 충족하는지 확인하고, 경험을 축적합니다.
- **calibrate**: 축적된 이탈을 감사합니다. 전제·제약이 흔들렸거나 concept과 코드가 어긋났을 때, 결과를 리포트로 정리해 shape 재진입 여부를 사용자에게 맡깁니다.

## 컴포넌트

loom은 `.loom/` 디렉토리에 다섯 가지 엔티티를 관리합니다.

### Concept

프로젝트의 도메인 엔티티를 정의하는 영속 문서입니다.
"이 시스템이 무엇을 다루는가"를 기술하며, 구현이 아닌 도메인에 집중합니다.
프로젝트의 공유 어휘(ubiquitous language) 역할을 합니다.

### Decision

concept의 형태에 영향을 준 결정의 맥락을 기록합니다.
왜 이렇게 결정했는지를 보존하여, 미래에 재검토할 수 있는 근거를 제공합니다.
각 decision은 독립적이고 자기완결적입니다.
유지 decision은 전제나 제약이 흔들렸음에도 기존 concept이나 결정을 유지한 근거를 보존합니다.

### Principle

반복된 경험에서 체득한, 판단의 근거가 되는 믿음입니다.
규칙이 아니라 근거로서, 같은 principle을 공유하면 누구든 같은 판단에 도달할 수 있게 합니다.
user scope와 project scope를 관통하는 유일한 엔티티입니다.

### Plan

설계를 실행 가능한 태스크와 의존관계로 구조화한 문서입니다.
shape의 산출물을 구현 단위로 분해하고, 진행 상황을 추적합니다.

### Scope

여러 `.loom/` 계층이 존재할 때 엔티티 간 우선순위를 결정하는 체계입니다.
user scope(`~/.claude/.loom/`)와 project scope(`.loom/`)가 있으며, 가장 가까운 scope가 우선합니다.

## 시작하기

### 사전 요구

- [aeira](https://github.com/grapgrap/aeira) -- `npm install -g @grapgrap/aeira`

### 설치

Claude Code의 플러그인으로 설치합니다.

```bash
claude plugin add grapgrap/loom
```

### 첫 사용

프로젝트에서 설계가 필요한 요구사항이 있을 때, shape부터 시작합니다.

```
/loom:shape 사용자 인증 기능을 추가하고 싶습니다
```

shape이 완료되면 plan으로 이어갈지, 바로 구현할지 안내합니다.

plan이 완료되면 task로 태스크를 실행합니다.

```
/loom:task
```

구현이 완료된 후에는 review로 검증합니다.

```
/loom:review
```

시간이 지나 문서와 코드 사이에 괴리가 느껴지면 calibrate로 교정합니다.

```
/loom:calibrate
```
