# Review

이번 세션의 산출물을 검토한다.

review는 메인이 직접 검토하지 않고 검토자(loom:reviewer)에게 위임한다. 같은 세션에서 산출물을 만든 주체는 그 합의에 매여 비판적 거리를 갖기 어렵기 때문이다. 메인은 산출물을 모아 검토자에게 넘기고, 검토자가 돌려준 비판을 사용자에게 종합한다. 직접 수정하지 않으며, 최종 판단은 사용자가 한다.

## Step 1 — 산출물 수집

이번 세션에 변경·생성된 파일 목록을 구성한다. `git status`로 변경된 파일을 확인하고, 세션에서 작업한 파일을 빠짐없이 모은다. 변경 내용은 검토자가 직접 읽으므로, 메인은 목록 구성까지만 한다. 대상 프로젝트의 `.loom` 경로를 확인한다.

## Step 2 — 검토 위임

loom:reviewer에게 Agent 도구로 검토를 위임한다.

- 위임 프롬프트에는 산출물 파일 목록과 `.loom` 경로만 전달한다. 검토 절차와 기준은 reviewer의 정의에 있으므로 다시 적지 않는다.
- 산출물을 요약하거나 해석하지 않는다. 메인의 해석이 검토자의 시야를 미리 좁히지 않도록, 산출물은 검토자가 직접 읽는다.

## Step 3 — 큐레이션

검토자가 반환한 비판을 사용자에게 종합한다.

- 검토자의 비판은 묵살하지 않고 모두 노출한다.
- 각 비판에 맥락을 더한다 — 이미 decision에서 다룬 사안인지, 다른 비판과 어떻게 얽히는지.
- 심각도(회귀·수정·관찰)로 정리한다. 검토자가 매긴 심각도는 잠정이며, 프로젝트 맥락을 아는 메인이 조정할 수 있다. 조정하더라도 비판 자체는 노출한다.

## 그래프 갱신

`.loom` 내 문서를 생성하거나 수정한 경우 `aeira sync -s {프로젝트의 .loom 경로}` 를 실행한다.

## Summary

review 완료 시 다음 형식으로 정리한다:

```
## Review Summary
- 검토 결과: [회귀 N건 / 수정 N건 / 관찰 N건]
- 다음 단계: [회귀가 필요하면 어느 단계로, 아니면 완료]
```
