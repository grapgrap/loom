# Meditate

축적된 원칙 전체를 감사하고 정제한다.

reflect가 원칙을 하나씩 축적하는 것이라면,
meditate는 축적된 전체를 조망하고 정리하는 것이다.

## 언제 실행하는가

- 사용자가 수동으로 호출할 때
- 원칙이 의미 있게 늘어났을 때 (시스템이 제안할 수 있으나, 실행은 사용자 결정)

## Step 1 — 전체 로드

`.loom/principles/`의 모든 파일을 읽는다. index.md부터 시작하여 모든 원칙 파일을 순회한다.

## Step 2 — 감사

각 원칙을 다음 기준으로 평가한다:

**유지 기준** — 다음 중 하나 이상을 충족하는가:
- 여러 review에서 반복적으로 유효성이 확인됨
- 위반 시 실제 문제가 발생한 이력이 있음
- 다른 원칙과 상호 보완적 관계

**제거/통합 대상**:
- 다른 원칙과 실질적으로 동일한 것을 말하는 원칙 → 통합
- 한 번도 review에서 참조되지 않은 원칙 → 제거 검토
- 너무 구체적이어서 특정 프로젝트에만 적용되는 원칙 → 제거
- 서로 충돌하는 원칙 → 하나를 선택하거나 조건을 명시

## Step 3 — 정제

감사 결과를 바탕으로:

1. 통합 가능한 원칙들을 합친다
2. 표현이 모호한 원칙을 구체화한다
3. 제거할 원칙을 사용자에게 제안한다 (직접 삭제하지 않는다)
4. `principles/index.md`를 업데이트한다

모든 변경은 사용자의 승인 후에 적용한다.
원칙은 사용자의 것이므로, 시스템이 임의로 삭제하지 않는다.

## Summary

meditate 완료 시 다음 형식으로 정리한다:

```
## Meditate Summary
- 감사 대상: [N개 원칙]
- 변경:
  - 통합: [A + B → C]
  - 구체화: [수정된 원칙]
  - 제거 제안: [근거와 함께]
- 원칙 수: [변경 전 N → 변경 후 M]
```
